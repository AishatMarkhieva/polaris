# POLARIS

Минималистичный пазл-платформер на **Python + Pygame**. Главная механика —
**смена полярности**: герой переключается между двумя цветами (синий/оранжевый),
и от этого мир вокруг меняется. Блоки твоего цвета — твёрдые (по ним идёшь),
блоки чужого цвета — сквозные (проходишь насквозь).

Игра вдохновлена *Polarity Cube* и подобными проектами, но построена так,
чтобы её было легко расширять и в будущем добавить кооператив.

## Как запустить

```bash
# 1) поставить зависимости (один раз)
pip install -r requirements.txt

# 2) запустить игру
python main.py
```

Нужен Python 3.10+ и pygame 2.x. Если pygame не ставится через requirements,
поставь напрямую: `pip install pygame`.

## Управление

| Действие             | Клавиатура         | Геймпад            |
|----------------------|--------------------|--------------------|
| Идти влево/вправо    | A / D              | левый стик / крест |
| Прыжок               | Пробел             | A (нижняя кнопка)  |
| Сменить полярность   | Shift (левый)      | X (левая кнопка)   |
| Рестарт уровня       | R                  | —                  |
| Подтвердить в меню   | Enter / Пробел     | Start / A          |
| Бесконечный режим    | G (на заставке)    | —                  |
| Выход                | Esc                | —                  |

Геймпад подхватывается автоматически, если подключён при запуске.

**Два режима:** на заставке `Enter` — кампания (4 рукотворных уровня),
`G` — бесконечный режим (уровни генерируются случайно и не кончаются).

### Подсказка по механике
Полярность нельзя сменить, если из-за этого ты окажешься внутри твёрдого
блока (чтобы не застрять). Поэтому переключайся, стоя на **нейтральных
серых блоках** (`#`) — они твёрдые всегда, на них всегда безопасно.

## Как расширять

### Добавить новый уровень
1. Создай `levels/level_05.txt`. Каждый символ — тайл:
   ```
   .  пусто            #  стена (всегда твёрдая)
   B  синий блок       O  оранжевый блок
   ^  шипы             *  кристалл
   @  старт игрока     E  выход
   ```
2. Впиши имя файла в список `LEVEL_FILES` в `src/game.py`.

Можно рисовать уровень прямо в текстовом редакторе или собирать его из
функций в `build_levels.py` (там это нагляднее и сетка всегда ровная).

### Подключить свои спрайты
Просто положи PNG в `assets/sprites/` с нужными именами — см.
`assets/sprites/README.md`. Код ничего менять не надо.

### Поменять фон уровня
Два способа (см. `assets/backgrounds/README.md`):
- **цвет:** в `settings.py` правь список `LEVEL_BACKGROUNDS` — там пары
  цветов (верх, низ) градиента на каждый уровень;
- **картинка:** положи `assets/backgrounds/level_01.png` (02, 03...) —
  игра возьмёт её вместо градиента для этого уровня.

### Бесконечный режим
Генератор случайных уровней живёт в `src/generator.py`. Уровни строятся
«шагающим строителем» так, чтобы путь всегда был проходим (ямы не шире
прыжка, ступеньки не выше прыжка, обязательная дорога — по нейтральным
блокам). Параметры частоты ям/шипов/кристаллов — прямо в этом файле.

### Подкрутить ощущение от игры
Всё в `settings.py`: скорость, сила прыжка, гравитация, трение, цвета.
Например, хочешь «потяжелее» — увеличь `GRAVITY` и `JUMP_VELOCITY`.

### Включить локальный кооператив (2 игрока на одном ПК)
Архитектура уже готова: игрок берёт «контроллер», а игроков может быть
несколько. В `src/game.py`, метод `_setup_controllers()`, раскомментируй
блок со вторым контроллером (игрок 2 — на стрелках или на втором геймпаде).
Камера сейчас следит за первым игроком — позже можно добавить общую камеру,
охватывающую обоих.

## Дорожная карта (идеи на будущее)

Близкие шаги:
- звук и музыка (pygame.mixer): прыжок, сбор кристалла, смена полярности;
- меню паузы и экран выбора уровня (это просто новые состояния в `game.py`);
- движущиеся платформы и кнопки/двери (новые типы тайлов);
- требовать собрать все кристаллы, чтобы открыть выход.

Средние шаги:
- локальный кооператив (см. выше) с пазлами «на двоих»: один синий,
  другой оранжевый, и они открывают путь друг другу;
- общая камера, которая держит в кадре обоих игроков.

Дальние шаги:
- сетевой мультиплеер. Самый простой путь — клиент-сервер на сокетах:
  один игрок хостит, передаёт состояние (позиции, полярности), второй
  присылает свой ввод. Логика игрока уже отделена от ввода, так что
  «сетевой контроллер» впишется в ту же абстракцию `input_manager.py`.

## Стек и почему именно он

**Python + Pygame** — лучший выбор для старта: минимум кода до первого
результата, всё на виду, легко понять, как устроена игра изнутри. Этого
с запасом хватит на 2D-платформер с уровнями, спрайтами и кооперативом.

Если позже захочется более «коробочных» возможностей (редактор уровней,
готовая физика, экспорт под Windows/Web/Android одной кнопкой) — присмотрись
к **Godot** (на нём, кстати, сделан Polarity Cube). Но переходить туда стоит
только когда упрёшься в потолок Pygame, а не раньше: здесь ты разберёшься
в самих основах геймдева, и это знание перенесётся в любой движок.
